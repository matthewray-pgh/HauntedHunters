# Expedition System

## Status
Core Gameplay Loop System

---

# Overview

Expeditions are the primary gameplay loop in Obsidian Frequency.

Instead of real-time manual control, players:
- select a team of Hunters
- choose equipment
- select a destination
- and initiate an automated mission simulation

---

# Core Loop

1. Select Hunters
2. Select Equipment
3. Choose Location
4. Launch Expedition
5. Receive Simulation Outcome
6. Return for Rewards and Analysis

---

# Design Philosophy

This system emphasizes:
- strategic preparation
- risk management
- collection progression
- and replayable simulation outcomes

It reduces mechanical complexity while maximizing:
- content scalability
- procedural outcomes
- and narrative variability

# Expedition Resolution

Expeditions resolve automatically based on:
- Hunter composition
- equipment synergy
- entity type
- environmental conditions
- and random anomaly factors

---

# Possible Outcomes

- Successful extraction
- Partial success
- Loss of equipment
- Hunter injury
- Hunter disappearance
- Rare entity discovery

---

# Return Phase

After expedition completion, players receive:
- collected entity data
- artifacts
- lore fragments
- upgrades
- and new unlocks

---

# Narrative Function

Expeditions generate:
- emergent storytelling
- unpredictable outcomes
- and evolving world states

---

# Design Intent

The player is not controlling moment-to-moment combat.

Instead, they are:
> managing risk and interpreting outcomes.
